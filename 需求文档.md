# 猫咪综合服务平台 — 需求文档

---

## 1. 项目概述

### 1.1 项目背景

随着宠物经济的快速发展，养猫用户对猫咪健康管理、社区交流、智能辅助等需求日益增长。本项目旨在构建一个集猫咪档案管理、社区互动、AI 智能问答、图像识别于一体的综合服务平台，同时提供完善的 PC 端管理后台，满足运营人员对内容、用户及数据的管理需求。

### 1.2 项目目标

- 为养猫用户提供便捷的移动端（H5）一站式服务
- 借助 AI 能力提升用户体验（智能问答、图像识别、内容分析）
- 为平台运营方提供高效的 PC 管理后台

### 1.3 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 通过 H5 端使用各项功能的养猫人群 |
| 平台管理员 | 通过 PC 端进行内容审核、用户管理等运营工作 |
| 超级管理员 | 拥有全部权限，负责系统配置与提示词管理 |

## 2. H5 用户端需求

### 2.1 猫咪档案

#### 功能描述
用户可为名下的猫咪创建和维护专属档案，记录猫咪的基本信息与健康数据。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| F-1.1 | 创建猫咪档案，填写名字、品种、性别、生日、体重等基本信息 | 高 |
| F-1.2 | 上传猫咪头像及多张照片 | 高 |
| F-1.3 | 支持多猫管理，用户可切换查看不同猫咪的档案 | 高 |
| F-1.4 | 记录疫苗接种、驱虫、绝育等健康事件，支持时间线展示 | 中 |
| F-1.5 | 编辑与删除档案 | 高 |

#### 非功能需求
- 档案数据需持久化存储，支持多端同步（iCloud/云端）
- 图片上传支持压缩处理，单张不超过 5MB

---

### 2.2 猫咪社区

#### 功能描述
提供用户之间分享猫咪日常、交流养猫经验的社区功能。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| F-2.1 | 发布帖子，支持文字 + 图片（最多 9 张）组合内容 | 高 |
| F-2.2 | 首页信息流展示，支持按时间/热度排序 | 高 |
| F-2.3 | 点赞、评论、收藏功能 | 高 |
| F-2.4 | 关注其他用户，查看关注用户的动态 | 中 |
| F-2.5 | 话题标签（#Tag）分类浏览 | 中 |
| F-2.6 | 举报违规内容 | 高 |

#### 非功能需求
- 信息流需支持下拉刷新与上拉加载更多（分页）
- 评论区支持二级回复

---

### 2.3 猫咪问答 AI（RAG）

#### 功能描述
基于检索增强生成（RAG）技术，为用户提供专业的猫咪养护、健康、行为等问题的智能问答服务。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| F-3.1 | 用户输入自然语言问题，AI 返回专业解答 | 高 |
| F-3.2 | 回答内容基于平台知识库（RAG），确保专业性与准确性 | 高 |
| F-3.3 | 支持多轮对话，保留上下文 | 高 |
| F-3.4 | 展示回答来源/参考文档（可选） | 低 |
| F-3.5 | 历史对话记录查看 | 中 |
| F-3.6 | 对回答进行点赞/踩反馈，用于模型优化 | 低 |

#### 非功能需求
- 单次问答响应时间 ≤ 5 秒（流式输出）
- 知识库内容由管理端维护（见 PC 端提示词/文件管理模块）

---

### 2.4 猫咪图像识别

#### 功能描述
用户上传猫咪照片，系统通过 AI 模型识别猫咪品种，并返回品种介绍及养护建议。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| F-4.1 | 用户从相册选择或实时拍照上传图片 | 高 |
| F-4.2 | AI 返回识别到的猫咪品种（含置信度） | 高 |
| F-4.3 | 展示该品种的基本介绍、性格特点、常见疾病、养护要点 | 高 |
| F-4.4 | 识别结果可一键保存至猫咪档案 | 中 |
| F-4.5 | 识别失败时给出友好提示 | 高 |

#### 非功能需求
- 图片识别响应时间 ≤ 8 秒
- 支持 JPG、PNG 格式，图片大小 ≤ 10MB

---

### 2.5 拍照打卡

#### 功能描述
用户每日为猫咪拍照打卡，记录猫咪成长轨迹，形成时间轴相册。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| F-5.1 | 每日打卡入口，拍照或选择照片完成打卡 | 高 |
| F-5.2 | 打卡日历视图，直观展示连续打卡天数 | 高 |
| F-5.3 | 打卡记录时间轴/相册浏览 | 高 |
| F-5.4 | 连续打卡成就/徽章激励机制 | 低 |
| F-5.5 | 打卡内容可选择同步发布至社区 | 中 |

---

### 2.6 文章展示与 AI 分析总结

#### 功能描述
平台提供专业养猫文章，AI 自动对长文进行摘要提炼，帮助用户快速获取核心信息。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| F-6.1 | 文章列表展示，支持分类筛选（健康、饮食、行为等） | 高 |
| F-6.2 | 文章详情页，支持图文混排 | 高 |
| F-6.3 | AI 自动生成文章摘要，展示在文章顶部 | 高 |
| F-6.4 | 用户可收藏文章 | 中 |
| F-6.5 | 文章阅读量、点赞数展示 | 低 |

#### 非功能需求
- AI 摘要在文章发布时由后台异步生成，不影响用户阅读体验

## 3. PC 管理端需求

### 3.1 档案管理

#### 功能描述
管理员可查看、搜索、管理平台上所有用户的猫咪档案数据。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| B-1.1 | 档案列表展示，支持按品种、性别、创建时间筛选 | 高 |
| B-1.2 | 搜索猫咪名称或主人用户名 | 高 |
| B-1.3 | 查看档案详情 | 高 |
| B-1.4 | 删除违规档案 | 中 |
| B-1.5 | 导出档案数据（CSV/Excel） | 低 |

---

### 3.2 用户管理

#### 功能描述
管理员对平台注册用户进行管理，包括查看用户信息、封禁/解封账号等操作。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| B-2.1 | 用户列表，展示昵称、注册时间、状态等信息 | 高 |
| B-2.2 | 搜索用户（按昵称、手机号、邮箱） | 高 |
| B-2.3 | 查看用户详情（发帖数、打卡数等） | 中 |
| B-2.4 | 封禁/解封用户账号，支持填写封禁原因 | 高 |
| B-2.5 | 用户角色权限分配 | 中 |

---

### 3.3 内容审核（AI 辅助）

#### 功能描述
对用户在社区发布的帖子、评论进行审核，AI 自动预审标记疑似违规内容，管理员进行人工复核。

#### 功能需求

| 编号 | 需求项 | 优先级 |
|------|--------|--------|
| B-3.1 | 待审核内容列表，展示内容摘要、发布时间、AI 风险评分 | 高 |
| B-3.2 | AI 自动检测违规内容（色情、暴力、广告等），标记风险等级 | 高 |
| B-3.3 | 管理员一键通过/拒绝内容，拒绝时填写原因 | 高 |
| B-3.4 | 批量审核操作 | 中 |
| B-3.5 | 审核日志记录（操作人、时间、结果） | 高 |
| B-3.6 | 用户举报内容的处理工作流 | 中 |

---

#### 功能描述
管理 AI 功能所使用的提示词模板，以及 RAG 知识库的文档文件，支持灵活配置 AI 行为。

#### 功能需求

| 编号    | 需求项                           | 优先级 |
| ----- | ----------------------------- | --- |
| B-4.1 | 提示词列表，按功能模块分类（问答、审核、摘要等）      | 高   |
| B-4.2 | 新增、编辑、删除提示词模板                 | 高   |
| B-4.3 | 提示词版本管理，支持回滚                  | 中   |
| B-4.4 | 知识库文件上传（PDF、TXT、Markdown 等格式） | 高   |
| B-4.5 | 文件解析状态展示（待处理/处理中/已完成/失败）      | 高   |
| B-4.6 | 删除知识库文件，同步清除对应向量数据            | 中   |

## 4. 非功能性需求

### 4.1 性能需求

- H5 端首屏加载时间 ≤ 3 秒（4G 网络环境）
- 接口响应时间 ≤ 1 秒（AI 接口除外）
- 系统支持并发用户数 ≥ 1000

### 4.2 安全需求

- 用户数据传输使用 HTTPS 加密
- 敏感信息（密码等）加密存储
- 管理端操作需记录审计日志
- 图片上传需进行内容安全检测

### 4.3 兼容性需求

- H5 端兼容主流移动浏览器（微信内置浏览器、Safari、Chrome）
- PC 管理端兼容 Chrome、Edge 最新两个版本

### 4.4 可用性需求

- 系统可用性 ≥ 99.5%
- 核心功能（档案、社区）支持离线查看缓存数据

---

## 5. 约束与假设

- 本项目为课程设计，优先实现核心功能，低优先级功能可在时间允许时实现
- AI 能力依赖第三方大模型 API（如 OpenAI、Claude 等），需考虑 API 费用与限流
- RAG 知识库初期由开发团队预置基础猫咪养护文档
- 图片存储使用云存储服务（如 OSS、S3）
