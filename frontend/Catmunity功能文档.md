# Catmunity 功能文档

## 项目概览

Catmunity 是一个**猫咪社区管理平台**，分为**用户端（移动端）**和**管理后台（桌面端）**两大部分。

### 技术栈

| 技术 | 用途 |
|------|------|
| Vue 3 (Composition API) | UI 框架 |
| TypeScript | 开发语言 |
| Vite | 构建工具 |
| Vue Router 4 | 客户端路由 |
| Vant 4 | 移动端 UI 组件库 |
| Ant Design Vue 4 | 管理后台 UI 组件库 |
| Axios | HTTP 请求 |
| ECharts 6 | 数据可视化图表 |
| Marked | Markdown 渲染（AI 对话） |
| postcss-px-to-viewport | 移动端 px→vw 自适应 |

## 一、用户端功能（移动端）

### 1.1 登录注册

#### 登录 (`/login`)
- 账号 + 密码登录
- 密码显示/隐藏切换
- 登录成功后存储 token 和 userId 到 localStorage，跳转至个人页
- 动画圆形背景装饰 + 猫咪 Logo 脉冲发光效果

#### 注册 (`/register`)
- 账号、邮箱、密码、确认密码、验证码注册表单
- 密码校验：8-16 位，必须包含字母和数字
- 邮箱验证码 60 秒倒计时防重复发送
- 注册成功自动登录并跳转首页

---

### 1.2 首页导航 (`/`)

- 顶部固定标题栏：Catmunity Logo + 退出登录按钮（确认对话框）
- **底部 Tab 栏（5 个标签）**：
    1. **档案** — 猫咪档案
    2. **社区** — 帖子广场
    3. **问答** — AI 智能问答
    4. **打卡** — 猫咪打卡
    5. **我的** — 个人中心

---

### 1.3 猫咪档案 (`/archive`)

- **双列网格**展示所有猫咪卡片
- 每张卡片展示：照片、名字、状态标签、品种、性别、年龄、体重
- **状态标签**：
    - 🟢 正常
    - 🟠 已领养
    - ⚪ 其他
- 性别显示："哥哥"（公）/ "妹妹"（母）
- 根据生日自动计算年龄（岁/月）
- 支持分页加载
- **详情弹窗**（底部弹出）：
    - 照片墙展示
    - 基本信息（生日、体重、状态、编号、创建/更新时间）
    - 健康事件时间线（彩色圆点 + 事件卡片）

---

### 1.4 社区广场 (`/community`)

#### 帖子信息流
- 帖子卡片展示：头像、用户名、相对时间、内容、图片网格（最多 3 列）、标签、操作栏
- **标签分类**：日常、好物分享、吐槽、搞笑 等
- **操作栏**：点赞 ❤️、评论 💬、收藏 ⭐
- 点赞/收藏支持**乐观更新**和**去重保护**
- 按话题标签搜索帖子
- 分页加载

#### 发布帖子
- 底部弹出表单
- 内容输入（最多 500 字，实时计数）
- 图片上传（最多 9 张，支持多选）
- 标签输入（最多 6 个，回车或按钮添加，点击删除）
- 提交按钮带加载状态
- 发布成功自动插入信息流顶部

#### 帖子详情与评论
- 底部弹出详情面板
- 照片墙 + 标签 + 完整内容
- **评论区**：
    - 嵌套评论树（父评论 + 子回复）
    - 回复支持 @提及
    - 删除自己的评论（确认对话框，递归统计子评论数）
    - 底部固定评论输入栏

---

### 1.5 AI 智能问答 (`/qa`)

- **会话列表**：展示所有历史对话，显示机器人图标、标题、时间
- **创建新对话** / **继续已有对话**
- **聊天窗口**（全屏弹窗）：
    - 用户消息气泡 + AI 回复气泡
    - AI 回复支持 **Markdown 渲染**（标题、列表、代码块、链接等）
    - XSS 安全过滤
    - **SSE 流式输出**：AI 回复实时逐字显示
    - **打字指示器**：动画跳动的点（AI 思考时显示）
    - **停止生成**按钮：可随时中断 AI 回复
    - **文件附件**：支持上传文件发送给 AI
    - 快捷键：`Ctrl + Enter` 发送
    - 使用 AbortController 取消进行中的请求
- 删除会话（确认对话框）

---

### 1.6 猫咪打卡 (`/camera`)

- **月历视图**：
    - 月份切换（左右箭头）
    - 星期标题行
    - 日期单元格：已打卡日期显示圆点标记，今天高亮
    - 图例：已打卡日 / 今日
- **统计栏**：当月打卡次数、连续打卡天数、是否满勤
- **打卡按钮**：大号"猫咪打卡"按钮

#### 打卡弹窗（底部弹出）
- 猫咪选择器（从猫咪列表中选择）
- 照片上传（支持相机拍摄）
- 备注文本输入
- "同步到社区"开关（自动创建社区帖子）
- 提交按钮

---

### 1.7 个人中心 (`/profile`)

- **个人信息头部**：头像、昵称、邮箱/简介、编辑按钮
- **统计行**：发帖数、关注数、粉丝数
- **菜单**：
    - "我的收藏"（带收藏数徽章）
- **我的收藏弹窗**：
    - 帖子卡片紧凑模式
    - 加载更多分页
    - 空状态提示
- **编辑资料弹窗**：
    - 账号、密码修改
    - 保存按钮带加载状态
- **退出登录**：确认对话框 → 清除 localStorage → 跳转登录页

## 二、管理后台功能（桌面端）

### 2.1 管理端登录 (`/admin/login`)

- **双栏布局**：左侧品牌面板（橙色渐变 + 装饰圆圈 + 功能介绍），右侧表单面板（白色卡片）
- 功能介绍列表：数据仪表盘、猫咪档案管理、社区内容、用户权限
- 表单：账号输入框、密码输入框、"记住账号"复选框、"忘记密码？"链接
- 登录按钮带渐变和阴影
- 登录成功存储 token、userId、userType、identifier，跳转仪表盘

---

### 2.2 管理布局 (`/admin`)

- **左侧边栏**（220px）：品牌头部（猫咪图标 + "Catmunity Admin Console"），5 个导航菜单项
    1. 仪表盘
    2. 猫咪档案
    3. 猫咪健康事件
    4. 知识库
    5. 用户管理
- 菜单选中态：橙色渐变高亮
- **顶部栏**：动态页面标题 + 用户下拉菜单（头像 + 退出登录）
- 退出登录清除所有 localStorage 并跳转管理端登录

---

### 2.3 仪表盘 (`/admin/dashboard`)

#### 概览卡片（6 个）
1. **总用户数**
2. **7 天活跃用户**
3. **总帖子数**
4. **猫咪档案数**
5. **总打卡数**
6. **待审核数**

每个卡片有图标、标签、数值和今日新增副文本。

#### 用户增长趋势图（ECharts 折线图）
- 维度切换：日 / 周 / 月
- 时间范围选择：7天 / 14天 / 30天
- 总增长量展示
- 平滑面积图带渐变填充
- 响应式自适应

#### 猫咪品种分布图（ECharts 环形图）
- 品种名称 + 数量 + 百分比
- 汇总文字：总猫咪数、品种数
- 多彩调色板，右侧图例

#### 健康事件提醒
- 未来 7 天内到期的事件提醒
- 卡片网格展示：事件类型标签、猫咪名称、标题、提醒日期（今天/临近/紧急标签）、描述
- 加载态和空状态

---

### 2.4 猫咪档案管理 (`/admin/cat`)

#### 搜索筛选
- 名称（模糊搜索）
- 品种（模糊搜索）
- 性别（下拉选择）
- 状态（下拉选择）
- 创建日期范围（日期选择器）
- 重置 / 查询 / 新增按钮

#### 数据表格
- 列：编号、照片（头像）、名称、品种、性别（粉/蓝标签）、生日、体重、状态（绿/橙/灰标签）、健康事件数（徽章）、创建时间、更新时间、操作（查看/编辑/删除）

#### 新增/编辑弹窗
- 名称（必填）、品种、性别（单选，必填）、状态（下拉，必填）、生日（日期选择）、体重（数字输入，步长 0.1）、照片（上传 + 预览，最多 9 张，自定义上传处理，校验图片类型和 5MB 大小限制）

#### 查看抽屉
- 照片墙（Ant Image 预览）
- 详细描述表格
- 健康事件时间线（彩色时间线）

#### 删除
- 确认对话框，删除最后一项自动调整分页

---

### 2.5 猫咪健康事件管理 (`/admin/health`)

- **猫咪选择器**：远程搜索下拉（按名称模糊搜索），每次加载 20 条
- **猫咪摘要卡片**：头像、名称、品种、事件数量
- **事件数据表格**：编号、事件类型（彩色标签）、标题、描述、事件日期、下次提醒日期、备注、操作（编辑/删除）

#### 新增/编辑弹窗
- 事件类型（下拉 + 彩色标签，必填）
- 标题（必填）
- 事件日期（必填）
- 下次提醒日期（可选）
- 描述（文本域，最多 200 字）
- 备注（最多 100 字）

#### 删除
- 确认对话框

---

### 2.6 知识库管理 (`/admin/knowledge`)

#### 搜索和数据表格
- 名称搜索、重置/查询/创建按钮
- 列：编号、名称（带书本图标）、描述、文档数（橙色标签）、分块数（青色标签）、创建时间、操作（详情/文档/删除）

#### 查看抽屉
- 描述信息表格
- 原始 JSON 数据查看器（暗色代码块）

#### 其他操作
- "创建数据集"和"删除"当前显示"功能开发中"提示
- 智能响应格式适配（支持 `array`、`{rows}`、`{list}`、`{data}`、`{records}` 多种响应形状）
- 灵活字段名解析（`name`/`title`/`datasetName`/`kbName` 均可解析为显示名称）

---

### 2.7 用户管理 (`/admin/user`)

#### 搜索筛选
- 关键词（账号/手机/邮箱）
- 用户类型（超管/管理员/用户）
- 账号状态（正常/封禁）
- 注册日期范围
- 重置 / 查询按钮

#### 数据表格
- 列：编号、用户（头像 + 标识符 + ID）、手机号、邮箱、用户类型（品红/橙/蓝标签）、状态（绿/红标签）、注册时间、最后活跃、操作（查看/封禁 或 解封）

#### 查看抽屉
- 用户头部（头像、标识符、类型/状态标签）
- 互动统计（4 个小卡片：帖子、打卡、关注、粉丝）
- 基本信息（编号、账号、手机、邮箱、类型、状态）
- 时间信息（注册时间、最后活跃、封禁时间）
- 封禁原因（如有）
- 底部操作按钮（封禁/解封、分配角色）

#### 封禁
- 弹窗含警告提示
- 用户编号（禁用）
- 封禁原因文本域（必填，最多 200 字）

#### 解封
- 确认对话框

#### 分配角色
- 用户编号（禁用）
- 类型单选（超管/管理员/用户，带彩色标签）
- 信息提示

## 四、架构设计

### 认证机制
- Token 存储在 localStorage
- Axios 请求拦截器自动注入 `token` 请求头
- 路由导航守卫：无 token 时重定向到登录页
- 白名单路由：`/login`、`/register`、`/admin/login`

### API 设计
- 统一响应格式：`{ code, msg, data }`（code=1 表示成功）
- Axios 响应拦截器自动解包 `data` 字段
- 统一错误处理：HTTP 错误码中文提示、业务错误 Toast 提示
- 401 自动跳转登录

### 状态管理
- 无全局状态库（无 Vuex/Pinia）
- 组件内使用 `ref`/`reactive`/`computed` 管理局部状态
- 数据变更后手动重新请求 API 刷新

### 响应式设计
- 移动端：Vant + postcss-px-to-viewport（375px 设计稿 → vw）
- 管理后台：Ant Design + px 单位（排除 vw 转换）

### Mock 系统
- 基于 Axios 请求拦截器的开发 Mock
- 通过 `VITE_USE_MOCK` 环境变量控制开关
- 当前 Mock 数据：社区帖子列表

### SSE 流式传输
- AI 对话使用原生 `fetch` + `ReadableStream` 处理 SSE
- 支持实时逐字显示和 AbortController 中断

## 六、API 模块总览

| 模块 | 端点 | 功能 |
|------|------|------|
| `authApi` | login/register/logout | 认证 |

| `adminUserApi` | user list/ban/role | 用户管理 |
| `aiApi` | dataset/sessions/message | AI 知识库 + SSE |
